Circuit Screenshare for UC

Bringing features to UC leveraging Circuit cPaaS

In the demo below, we have two users sharing their screen over Circuit backend using the Circuit JS SDK.
An "implicit" application is created on beta.circuit.com with clientID : 4b83dfe402644d3bb921e79049198802.
To skip the auth. steps on either side, we generated the Access Token by following the login flow once and storing the accessToken (_client.accessToken) for each user.
In the left pane, we are connecting with user #1 samantha.faw@unify.com. AccesToken is 1e6d8b9bff5646c7b1e85e2fd9cea273.
In the right pane, we are connecting with user #1 samantha.halfon@gmail.com. AccessToken is 24fff8b771e54b86b2cbc16ec442ec89.
Upon page load, each frame performs its logon sequence for the user.
A conference exists : 68949b94-402e-430c-82d4-cfc6503065a0
Each user can initiate the screen share, it will start the call in that conversation and start screensharing.

Screenshare requires the Circuit extension for Chrome
